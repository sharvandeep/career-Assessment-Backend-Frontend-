PROBLEM STATEMENT: FSAD-PS30
Create a web-based career assessment tool for students

Develop a platform that offers career assessments, personality tests, and skills
evaluations. The tool should help students understand their strengths and 
preferences and provide recommendations for suitable career paths.

Admin: Manage assessment tools, update test data, provide career recommendations.
User (Student): Take career assessments, view results, explore career options.

================================================================================
                    SKILL-CAREER MAPPING SYSTEM ← NEW SECTION!
================================================================================

This section explains how skill categories flow through the system to generate
career recommendations for students.

--------------------------------------------------------------------------------
                         DATA FLOW ARCHITECTURE
--------------------------------------------------------------------------------

┌─────────────────┐     ┌──────────────────┐     ┌────────────────────┐
│  BranchSkill    │────>│  Faculty Creates │────>│  Student Takes     │
│  Entity         │     │  Assessment      │     │  Assessment        │
│  (C, Java,      │     │  (Tags questions │     │  (Responses saved) │
│   Python...)    │     │   with skills)   │     │                    │
└─────────────────┘     └──────────────────┘     └────────────────────┘
                                                          │
                                                          ▼
┌─────────────────┐     ┌──────────────────┐     ┌────────────────────┐
│  Career         │────>│  Match Algorithm │<────│  Skill Analysis    │
│  Recommendations│     │  (Compare skills │     │  (Calculate % per  │
│  (Sorted by %)  │     │   with career)   │     │   skill category)  │
└─────────────────┘     └──────────────────┘     └────────────────────┘

--------------------------------------------------------------------------------
                         STEP-BY-STEP PROCESS
--------------------------------------------------------------------------------

1. BRANCH SKILLS (BranchSkillEntity) - TOTAL: 43 SKILLS
   ─────────────────────────────────────────────────────
   Each branch has predefined skill categories:
   
   CSE Branch (8):    C, Java, Python, Data Structures, DBMS, 
                      Web Development, Operating Systems, Computer Networks
   ECE Branch (7):    Digital Electronics, Analog Electronics, Signals & Systems,
                      Embedded Systems, VLSI, Communication Systems, Microprocessors
   CSIT Branch (7):   C, Java, Python, Computer Networks, Cybersecurity,
                      Cloud Computing, Linux Administration
   AIDS Branch (7):   Python, Machine Learning, Statistics, Data Analysis,
                      Deep Learning, Big Data, Data Visualization
   MECH Branch (7):   Thermodynamics, Mechanics, CAD/CAM, Manufacturing,
                      Fluid Mechanics, Material Science, Robotics
   BIO-TECH (7):      Biotechnology, Genetics, Biochemistry, Microbiology,
                      Bioinformatics, Cell Biology, Immunology

2. FACULTY CREATES ASSESSMENT
   ────────────────────────────
   - Faculty selects skill category for each question from dropdown
   - Dropdown populated via: GET /skills/{branchId}
   - Each question is tagged with ONE skill category
   - Example: "What is polymorphism?" → skillCategory: "Java"

3. STUDENT TAKES ASSESSMENT
   ────────────────────────────
   - Student answers questions
   - Each response stored in StudentResponseEntity
   - Links: studentId, questionId, assessmentId, selectedAnswer

4. SKILL ANALYSIS (GET /student/skill-analysis/{studentId})
   ────────────────────────────────────────────────────────
   Backend calculates performance per skill category:
   
   Response Format:
   [
     { "skill": "Java", "correct": 8, "total": 10, "percentage": 80.0 },
     { "skill": "Python", "correct": 6, "total": 8, "percentage": 75.0 },
     { "skill": "DBMS", "correct": 5, "total": 10, "percentage": 50.0 }
   ]

5. ADMIN CREATES CAREER PATHS
   ────────────────────────────
   Admin defines career paths with required skills:
   
   Example Career Path:
   - careerName: "Backend Developer"
   - description: "Build server-side applications"
   - requiredSkills: "Java, Python, DBMS"  (comma-separated)
   - minimumSkillPercentage: 60
   - branchId: null (all branches) or specific branch ID

6. CAREER RECOMMENDATIONS (GET /student/career-recommendations/{studentId})
   ─────────────────────────────────────────────────────────────────────────
   Backend matching algorithm:
   
   a) Get student's skill percentages from all assessments
   b) For each career path:
      - Extract required skills (split by comma)
      - Check student's percentage in each required skill
      - Calculate average match percentage
   c) Assign match level:
      - EXCELLENT: >= 80%
      - GOOD: >= 60%
      - FAIR: >= 40%
      - DEVELOPING: < 40%
   d) Return sorted by matchPercentage (highest first)

   Response Format:
   [
     {
       "id": 1,
       "careerName": "Backend Developer",
       "description": "Build server-side applications",
       "requiredSkills": "Java, Python, DBMS",
       "matchPercentage": 68.33,
       "matchLevel": "GOOD"
     }
   ]

--------------------------------------------------------------------------------
                         CRITICAL MAPPING RULES
--------------------------------------------------------------------------------

⚠️  IMPORTANT: Skill names MUST be consistent across:

    1. BranchSkillEntity.skillName    (e.g., "Java")
    2. QuestionEntity.skillCategory   (e.g., "Java") 
    3. CareerPathEntity.requiredSkills (e.g., "Java, Python")

    If Admin creates career with "JavaScript" but branch skills have "Java",
    the matching will fail! Names must match EXACTLY (case-sensitive).
